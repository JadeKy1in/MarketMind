# 智能量化投资分析系统 - 项目概要

## 项目标识

- 项目名称: SignalFoundry - 智能量化投资分析系统
- 项目目录: `projects/robinhood/`
- 运行环境: Python 3.11+
- 核心 LLM: DeepSeek（唯一指定推理后端）

## 核心目标

构建一套基于反身性理论（Reflexivity）与事件驱动的低频量化分析工具，追求每年仅执行 3 至 5 次高回报投资，单次预期收益率 > 20%。

系统采用"自上而下"（Top-down）逻辑推演而非纯数据驱动，通过四维共振分析引擎对基本面、技术面、事件驱动力及市场情绪进行交叉验证，仅在至少三个维度产生强烈共振时触发交易信号。

## 不可协商的硬性约束（系统天条）

### 天条 1: UI/输出纪律
所有代码注释、控制台输出、分析报告及系统日志中，绝对禁止出现表情符号（Emojis）或任何无意义的装饰性字符。输出格式遵循 ASCII-only 纯文本结构化 Markdown。

### 天条 2: 物理隔离纪律
本系统是**分析工具**，不是自动交易机器人。系统决不允许直接调用券商 API 执行买卖。所有关于资金状况的测算，必须基于手动配置的本地 JSON 文件（"账户状态人工输入协议"）。

### 天条 3: 反拟合纪律
摒弃价格序列的高频回测拟合，坚持"自上而下"逻辑推演与四维共振打分的底层逻辑。价格序列数据仅作为"买入时机"的过滤参考，不能作为信号生成的根本依据。

## 成功度量标准

- 系统年化信号数量: 3-5 个（非牛市年份可更低）
- 单信号平均收益率目标: >20%
- 最大回撤容忍度: <15%
- 空仓期间占比容忍度: 70%（不惧空仓，耐心等待）
- 四维共振阈值: 至少 3/4 维度共振方可发出信号

## 开发方法论

沿用当前工作区的 SkillFoundry 工具链与智能体编排范式：
- `.clinerules` 的五大矩阵纪律体系全面适用
- SPARC 认知循环（Specification -> Pseudocode -> Architecture -> Refinement -> Completion）
- Memory Bank 协议双向同步
- PM 治理协议（铁律一至五）
- 开发阶段利用高级工具链，最终产物为纯 Python 轻量级脚本
