# 待开发: 多模态聊天输入——用户在Gate讨论中提交投行材料

**状态**: PENDING — 设计阶段  
**创建日期**: 2026-05-17

## 场景

用户在三个Gate的讨论中，可能收到投行朋友给的材料（PDF/图片/文档/文字），想发给主AI一起分析。

## 现状

- `gateway/multimodal_adapter.py` 已实现（Gemini Vision + Tesseract + pdfplumber）
- 但它是影子生态的 `ExternalObservation` 管道——不是主AI聊天输入
- CLI 模式下无法拖动文件

## 设计方案

### 输入方式

```
用户: "我朋友发来一份PDF，在 data/incoming/gs_2026_outlook.pdf"
      "帮我看看里面说了什么"

或直接粘贴文本:

用户: "我朋友说: [粘贴投行建议原文]"
```

### 处理流程

```
用户提交文件/文本
      │
      ▼
MultimodalAdapter.extract_text()
  ├─ PDF → pdfplumber 提取文字
  ├─ 图片 → Gemini Vision OCR
  └─ 纯文本 → 直接传入
      │
      ▼
注入当前 Gate 的讨论上下文
      │
      ▼
主AI分析并回复:
  "这份高盛报告的核心观点是...
   与我们今天的分析有以下异同: ...  
   需要核实的数据点: ..."
```

### 集成点

三个Gate都可以接收外部材料——用户在任意阶段提交，AI在当前讨论上下文中分析。

### 需要新增的模块

| 模块 | 功能 |
|------|------|
| `ui/chat_input.py` | 聊天输入框，支持文件拖放或路径输入 |
| `gateway/chat_context.py` | 管理多轮对话上下文，注入外部材料 |
| `app.py` 修改 | `run_daily()` 加入交互循环 |

### 独立判断原则（关键）

**主AI不得一味认同用户提交的外部材料。** 投行材料可能带有推销目的、选择性引用数据、或遗漏关键反证。

处理外部材料时：
1. 提取材料中的**可验证声明**（价格、比率、日期、预测值）
2. 对照**自己的数据源**逐一验证（FRED/EIA/yfinance等）
3. 标记：`[已验证✓]` / `[无法验证⚠️]` / `[与数据矛盾✗]`
4. 如果材料结论与AI独立分析相悖——**直说，不迎合**
5. 外部材料显式标记 `[外部投行材料]` 与AI自主分析区分

### 安全考虑

- 文件路径限制在 `data/incoming/` 目录内
- PDF 不执行任何脚本
- 图片先 OCR，不直接展示
