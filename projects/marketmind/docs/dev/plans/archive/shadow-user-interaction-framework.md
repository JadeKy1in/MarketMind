# Gate 2 影子-用户-主管道交互框架

**Date**: 2026-05-20
**问题**: Elite 影子在 Gate 2 如何与用户交流？影子观点和主管道观点之间如何避免交叉污染？
**目标**: 设计一个结构化交互框架，既允许用户充分利用两边的独立分析，又不破坏独立性。

---

## 1. 核心原则

### 1.1 独立性优先

影子和主管道之间的独立性是系统设计的基石。一旦影子知道主管道的结论（或反过来），分析就不再独立——锚定效应会破坏两者的价值。

### 1.2 用户是唯一桥梁

只有用户同时看到两边。用户决定在多大程度上让两边观点交叉。系统不自动传递任何结论。

### 1.3 结构化提问 > 开放式对话

不提供"随便聊聊"式的对话。提供一套结构化问题框架，帮助用户系统地审视两边的分析，补足盲点。

## 3. 污染风险评估与缓解

### 3.1 污染场景

| 场景 | 风险等级 | 是否允许 | 缓解措施 |
|------|:--:|:--:|------|
| 用户告诉主管道"影子 A 说应该做空" | HIGH | ⚠️ 允许但不推荐 | 系统提示："注意：分享其他分析师的结论可能影响本分析师的独立判断" |
| 用户告诉影子"主管道认为应该做多" | HIGH | ⚠️ 允许但不推荐 | 同上 |
| 用户以假设形式提问"如果有人觉得X，你怎么看" | MEDIUM | ✅ 推荐 | 不透露来源，以假说形式探讨 |
| 用户使用框架问题直接提问 | LOW | ✅ 推荐 | 问题设计不携带其他分析师的信息 |
| 系统自动分享结论 | CRITICAL | ❌ 禁止 | 系统不自动传递任何一方的结论给对方 |

### 3.2 设计决策

**用户可以做任何事——因为用户是最终决策者。** 系统不阻止用户分享信息，但：

1. 提供结构化问题框架（引导用户以低污染方式提问）
2. 所有交叉引用被记录在审计日志中
3. 当用户即将分享对方结论时，显示温和提醒

## 4. 结构化问题框架

### 4.1 为什么需要框架

- 开放式对话容易让用户陷入确认偏误（只问支持自己观点的问题）
- 用户可能忽略关键的场景——"万一错了怎么办？"
- 影子/主管道的回答质量高度依赖于问题质量

### 4.2 预设问题清单

**基础验证（每次必问）:**

| # | 问题 | 目的 |
|:--:|------|------|
| Q1 | "你这个判断中最脆弱的一环是什么？" | 迫使 AI 暴露自己推理的弱点 |
| Q2 | "什么信号或事件会让你改变这个判断？" | 明确证伪条件 |
| Q3 | "你在这个判断上的置信度，过去类似判断的准确率是多少？" | 校准置信度（Brier Score） |

**风险探查:**

| # | 问题 | 目的 |
|:--:|------|------|
| Q4 | "如果市场向相反方向移动 5%，你怎么办？" | 压力测试 |
| Q5 | "你最担心但觉得发生概率低的尾部风险是什么？" | 暴露隐藏风险 |
| Q6 | "有没有你希望看到但还没看到的数据？如果有，是什么？" | 发现信息缺口 → 自动创建 Pending Signal |

**时间与节奏:**

| # | 问题 | 目的 |
|:--:|------|------|
| Q7 | "这个判断的时间窗口是多长？什么时候算'错了'？" | 明确定义退出条件 |
| Q8 | "未来 2 周内有什么事件可能改变这个判断？" | 前瞻性风险识别 |
| Q9 | "现在是入场的最佳时机，还是应该再等什么信号？" | 择时判断 |

**替代场景:**

| # | 问题 | 目的 |
|:--:|------|------|
| Q10 | "如果你必须为相反方向辩护，你会怎么说？" | 强制考虑反面 |
| Q11 | "假设你的核心假设错了，第二可能的情况是什么？" | 替代场景思考 |
| Q12 | "有什么你无法量化但让你不安的因素？" | 定性风险评估 |

### 4.3 框架使用方式

用户可以选择：
- A) 一键发送所有 12 个问题 → AI 逐一回答
- B) 选择部分问题发送
- C) 手动输入自己的问题
- D) 混合模式（预设 + 手动跟进）

## 5. Elite 影子交互

### 5.1 谁能和用户交流

只有达到 **Elite** 等级的影子才会出现在 Gate 2 对话界面中。

| 等级 | 能否在 Gate 2 对话 |
|------|:--:|
| Elite | ✅ 可以 |
| Excellent | ❌ 不可（只展示摘要结论） |
| Watch/Endangered | ❌ 不可（只排名可见） |

这创造了一个正向激励：影子不只是为排名而竞争——排名靠前的影子获得与用户交流的"话语权"。

### 5.2 Elite 影子的交互范围

Elite 影子在 Gate 2 中只讨论**自己领域内的内容**：

| Elite 影子 | 可以讨论的话题 |
|------|------|
| Bullion Broker | 贵金属、通胀、美元、央行购金 |
| Cycle Reader | 宏观周期、跨资产配置、体制判断 |
| Fade Master | 共识极端、逆向机会、情绪信号 |
| ... | ... |

如果用户问 Bullion Broker 关于科技股的问题，它应该回答："这不是我的领域。建议咨询 Silicon Oracle（如果它是 Elite 的话）"。

### 5.3 影子不参与最终决策

Elite 影子在 Gate 2 的角色：
- ✅ 解释自己的分析和判断
- ✅ 回答用户的质疑
- ✅ 指出自己分析中的不确定性
- ❌ 不投票、不融合、不参与主管道的最终决策
- ❌ 不知道主管道得出了什么结论（除非用户主动告诉它）

## 6. 交叉验证协议

### 6.1 用户驱动的交叉验证

用户如果想对比两边的观点：

**推荐做法（低污染）**:
1. 用户先用框架问题分别问两边（不透露对方的结论）
2. 用户自己对比两边的回答
3. 如果发现矛盾，用户可以以**假说形式**追问：
   - "假设有人认为 [对立观点]，你怎么回应？"（不透露来源）
   - "你的分析中哪个环节最可能被质疑？"

**不推荐但允许的做法（中污染）**:
- 直接告诉一方对方的结论，要求评论
- 系统会记录这类交叉引用，并在审计日志中标注

### 6.2 审计日志

所有 Gate 2 的交互都被记录：

```json
{
  "date": "2026-05-20",
  "user_actions": [
    {"action": "view_main_conclusion", "timestamp": "..."},
    {"action": "view_shadow_summary", "timestamp": "..."},
    {"action": "question_elite_shadow", "shadow_id": "expert:macro:cycle_reader",
     "question_type": "preset_Q2", "timestamp": "..."},

  ],
  "final_decision": {"direction": "long", "ticker": "XLK", "user_override": false}
}
```

## 8. 时间线

```
Gate 2 开启
    │
    ▼
Phase 1: 独立展示（5 分钟）
    ├── 用户看主管道结论
    ├── 用户看影子研究摘要
    └── 系统不自动交叉引用
    │
    ▼
Phase 2: 结构化提问（用户驱动，不限时）
    ├── 用户使用框架问题分别问两边
    ├── 用户对比回答
    └── 用户可能发现矛盾或盲点
    │
    ▼
Phase 3: 交叉验证（用户可选）
    ├── 用户以假说形式追问
    ├── 系统记录交叉引用
    └── 用户形成自己的判断
    │
    ▼
Phase 4: 决策
    ├── 用户确认/修改/推翻主管道结论
    ├── 系统记录最终决策 + 影响因子
    └── Gate 2 关闭 → 进入 Gate 3
```

## 9. 外部研究待确认

以下问题需要通过外网研究来验证和优化：

1. **结构化问题框架**: 最好的投资决策辅助工具（如 Bloomberg / FactSet / Eikon 的研究管理功能）使用什么样的结构化框架？
2. **分析师独立性**: 多策略基金如何保持独立 PM 之间的信息隔离？Millennium/Citadel 的内部信息隔离机制？
3. **人机交互**: 最有效的 AI-辅助投资决策交互模式是什么？对话式 vs 仪表盘式 vs 混合式？
4. **Elite 交互权限**: 只让排名靠前的影子参与交流——这在多策略基金中是否有类似的"track record 门槛"机制？
5. **污染控制**: 学术文献中关于多模型独立性的保持方案有哪些？
