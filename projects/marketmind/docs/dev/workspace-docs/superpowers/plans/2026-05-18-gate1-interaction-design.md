# Gate 1 主AI与用户交互方案

**日期**: 2026-05-18 | **基于**: 4 份外网研究 | **待用户审批 → 红方审核**

---

## 一、Gate 1 在管道中的位置

```
用户回到电脑前 ← 此时 AI 已完成 Stage 1-3 全部分析
      │
  ┌───▼──────────────────────────────────────────┐
  │              Gate 1: 方向确认                 │
  │                                              │
  │  1. Scout Monitor 摘要（30秒扫读）            │
  │  2. AI 呈现 N 个投资方向（假设卡片）           │
  │  3. 用户浏览/追问/提出新方向                  │
  │  4. 确定方向 → 进入 Stage 4-8                │
  └──────────────────────────────────────────────┘
```

## 二、Gate 1 对话流程

### 第一步: Scout Monitor 摘要

AI 首先呈现当日信息源状态（30秒扫读，研究5建议"先同步信息基线"）

```
AI: "今日信息源正常。35源中33工作，共587条新闻。
    3条重要提示: [无/有新增失效源/某PRIMARY源离线]"
```

颜色标记: 🟢全部正常 / 🟡有降级 / 🔴PRIMARY源离线

### 第二步: 呈现投资方向（假设卡片）

AI 呈现 HVR 调查循环产出的假设。每个假设用**等权重卡片**展示，AI 不做推荐排序（研究1: 防锚定）：

```
┌─────────────────────────────────────────────────────┐
│ 方向 A: EUR 看涨                       置信度 0.81  │
│                                                     │
│ 核心逻辑: ECB 鹰派措辞 + 德国 PMI 超预期            │
│         + EUR/USD 处于低位 → 存在上行空间            │
│                                                     │
│ ✅ 支持证据 (4层验证):                               │
│   Layer1 市场定价: 利率期货仅定价20%加息概率         │
│   Layer2 基本面: 核心CPI 3.2% > 2%目标              │
│   Layer3 多源: FT+Reuters+Bloomberg 3源一致          │
│   Layer4 历史: 2022年类似措辞后EUR涨4%               │
│                                                     │
│ ❌ 反对意见 (强制看空):                              │
│   ECB 可能只是口头干预，实际不加息                   │
│   看空置信度 0.45，未达降级阈值                      │
│                                                     │
│ ⚠️ 如果这个方向错了:                                │
│   ECB 下次会议明确鸽派 → EUR 可能继续走弱            │
│                                                     │
│ 风险: 中等 | 时间窗口: 2-4周                         │
└─────────────────────────────────────────────────────┘
```

（每个假设一张卡片，最多5张）

### 第三步: 引导问题（反苏格拉底原则）

AI 呈现 ≤3 个引导问题，**不引导方向**，只帮用户组织思考（研究3）：

```
"这里有3个方向。你想:
  A) 先看哪个方向的完整逻辑链？
  B) 对比两个方向的反对意见？
  C) 或者你有完全不同的方向想讨论？"
```

**原则**（研究2: 80/10/10）: 80% 用户发言时间，10% 数据，10% AI 输入。

### 第四步: 用户主导探索

用户选择后，AI 展开对应内容。用户可以：
- **追问细节**: "EUR 那个，反对意见为什么只有 0.45？"
- **交叉对比**: "EUR 和 TLT 的反对意见哪个更强？"
- **提出新方向**: "我朋友说应该投大豆，你怎么看？"

### 切换到新方向

用户提出新主题时（研究1: Parking Lot 模式 + Pivot 语言）：

```
AI: "大豆是个我还没分析的方向。给我 ~2分钟查数据和验证。
    你先看看现有的 EUR 和 TLT 分析？还是等大豆结果出来一起看？"
```

如果材料复杂（研究4: 复杂度分诊 T0-T3），AI 评估时间后主动说:

```
AI: "你这份PDF包含投行的详细模型数据。
    我需要 ~5分钟做完整验证（识别可验证声明→对照FRED/EIA→交叉验证）。
    建议你去喝杯咖啡，5分钟后回来我们详细讨论。"
```

## 三、白盒归档

Gate 1 的完整对话以 **JSONL + Markdown** 双格式归档（研究5）：

```
data/archive/2026/05/18/
├── analysis/           ← Stage 1-3 自动分析（已有）
├── decisions/          ← Stage 8 决策（已有）
├── review/             ← Stage 6-7 审查（已有）
├── gates/              ← 新增
│   ├── gate1_conversation.jsonl   ← 结构化对话日志
│   ├── gate1_conversation.md      ← 人类可读版本
│   ├── gate1_decision.json        ← 最终方向决定 + 放弃的方向
│   ├── gate2_conversation.jsonl
│   └── ...
```

JSONL 每行一个 turn:
```json
{"turn": 1, "speaker": "AI", "type": "hypothesis_card", "direction": "EUR_long", "confidence": 0.81}
{"turn": 2, "speaker": "USER", "type": "question", "text": "反对意见为什么只有0.45？"}
{"turn": 3, "speaker": "AI", "type": "bear_case_detail", "direction": "EUR_long", ...}
```

用途: AI 复盘时读取，红方审计时验证。

## 四、三种会话模式

| 模式 | 用户时间 | Gate 1 行为 |
|------|:---:|------|
| **完整** | 充裕 | 全部假设卡片 + ≤3引导问题 |
| **快速** | 2-3min | 仅呈现最高置信度方向 + 1个问题: "选这个还是你有别的？" |
| **追赶** | N天未登录 | 跳过 Gate 1，呈现 N 天变化摘要 |
