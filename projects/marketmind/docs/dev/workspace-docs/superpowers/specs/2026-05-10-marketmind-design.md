# MarketMind — AI 人机协作投资分析工作站

**版本**: v1.2 | **日期**: 2026-05-11 | **状态**: 六方红队审计修订完毕

---

## 1. 产品概览

### 1.1 一句话定义

基于 DeepSeek V4（1M 上下文 + Max 推理）的人机协作投资分析工作站。AI 负责信息整合、多层分析、红方质疑和影子进化；人负责方向确认和最终决策。所有分析原材料和决策记录全本地归档。

### 1.2 核心目标

1. **提高胜率，而非最大化收益**。识别高质量信号，在趋势确立后参与、登顶前或趋势反转早期撤离。追求 60%+ 胜率而非极端收益。
2. **人机各司其职**。AI 做信息收集、多维度分析、信号共振、红方质疑、影子实验；人做方向确认和最终决策。
3. **每天比昨天更好**。三层复盘（日/月/季）+ 影子竞争进化 + 认知库积累，确保方法论持续迭代。
4. **全本地归档**。所有原材料、中间分析、决策记录、复盘结论完整保存，支持任意时间点的回溯审计。

### 1.3 不做的事

- 不连接任何券商 API（物理隔离原则）
- 不做实时高频交易
- 不做多用户系统

## 2. 七大系统法则

| 法则 | 内容 |
|------|------|
| **Law 1 — 输出规范** | 最终报告 ASCII-only，无 emoji。代码注释和界面文字例外。 |
| **Law 2 — 物理隔离** | 绝不连接券商 API。Robinhood 平台操作用户手动执行。覆盖 Robinhood 所有可交易标的（股票/ETF/期权/加密货币）。 |
| **Law 3 — 反过拟合** | 价格数据是时机过滤器，不是信号源。信号共振需经 Deflated Sharpe Ratio 和 CSCV/PBO 统计框架验证（PBO > 10% 视为无信号）。不暴力调参。空仓是有效输出。 |
| **Law 4 — 模型路由** | Pro + Flash 均使用 1M 上下文 + reasoning_effort="max"。与用户交互/方向判断/最终决策 → Pro。数据采集/粗筛分类/影子日常分析/格式化 → Flash。Python 直接处理 → 不用模型。 |
| **Law 5 — 分析资源纪律** | 机会发现资源不因持仓状态削减。持仓管理是额外追加的独立预算。在投资机会和亏损风险面前不吝啬 AI 调用。 |
| **Law 6 — 人在回路** | AI 不下单、不决策。所有方法论变更需人类批准。影子可自由实验但不出沙盒。 |
| **Law 7 — 数据诚信（Data Integrity）** | 绝对禁止捏造数据。所有 AI 输出中的数字声明（价格、比率、百分比、日期、金额）必须有可追溯的数据来源。无法获取数据时必须诚实报告"数据不可用"，不得猜测、模拟、或编造。违反此法则的影子或 Agent 立即隔离，三次违规终身销毁。此法则优先级高于所有其他系统目标（包括胜率、排名、配额）。 |

## 4. 分析管道：三层并行独立架构

**核心原则**：Layer 3 独立于 Layer 1-2 运行，各自只看到原始数据，不继承上游结论。Layer 1-2 可并行，Layer 3 完全独立。差异由 Pro 最终协调。

### 4.0 信号共振：统计验证框架

取代简单的 ≥3/4 维度计数。每个信号必须通过：

- **Deflated Sharpe Ratio (DSR)**：校正多重信号比较的选择偏误
- **CSCV/PBO**：概率化回测过拟合检测。PBO > 10% 的信号视为"无信号"
- **前向验证**：30 天样本外验证，表现 ≥ 评估期 50% 才通过

### 4.0.1 "今天不交易"信号生成器

与决策生成器对等的独立组件，以同等分析深度产出"什么都不做"的最佳论证。在门禁 3 与买入/卖出建议并排呈现。在熊市/高 VIX 环境中具有结构性优势（更低的"胜出"门槛）。

### 4.1 Layer 1：事件 + 情绪 + 人物（回答：市场在讲什么故事？被定价了吗？）

#### 4.1.1 事件扫描

- **A-E 分级**：货币政策(A) > 公司行动(B) > 监管政策(C) > 地缘(D) > 宏观数据(E)
- **2×2 矩阵**：意外程度(高/低) × 市场规模(大/小) → 核心机会/顺势机会/套利机会/观察跳过
- **Price-in 评分**：比对期权隐含波动 + 事件前价格漂移 + 分析师共识 → 市场定价程度
- **级联追踪**：事件 → 一阶(直接) → 二阶(资本重配) → 三阶(结构性转变)，标记"级联枢纽"

#### 4.1.2 情绪分析（结构化输出）

- 综合情绪方向 + 程度
- 指标支撑（VIX/Put-Call/美债/黄金/资金流向），每条 = 数据 + 方向 + 一句话解读
- 情绪 vs 注意力分离（高情绪 = 可能反转；高注意力 = 可能持续）
- 专家筛选信号（按历史准确率筛选"真专家"和"连错王"作为反向指标）
- 制度相对意外（央行官员说了什么 vs 当前经济数据预测他会说什么）

#### 4.1.3 关键人物言行

三级验证链：**能力 → 意愿 → 市场认可**

| 级 | 检查 | 核心问题 |
|----|------|---------|
| ① 能力 | 他有执行这个主张的权力/资源吗？ | 行政命令 vs 需要国会？ |
| ② 意愿 | 政治表演还是真实意图？ | 中期选举？个人利益？TACO 模式？ |
| ③ 市场认可 | 市场相信吗？ | 相关资产 30 分钟内波动？脉冲还是持续？ |

政治人物交易信号不对称原则：**卖出信号 > 买入信号**。延迟披露检测：41%的延迟申报交易有更高异常收益——拖延本身就是信号。

马赛克拼图规则：草根信息只能作为引子，必须经 3 条独立来源验证才可升级为分析信号。

#### 4.1.4 尾部风险扫描

- 跨资产隐含相关性飙升
- VIX 期限结构倒挂
- 波动率聚集检测（Fibonacci 窗口）
- ≥2 个触发 → 当日风险评级提升

### 4.2 Layer 2：基本面分析（回答：哪些标的真正受益？市场忽略了什么？）

五层递进，每层经红方质疑：

| 层次 | 做什么 | 产出 |
|------|--------|------|
| L2.1 宏观象限 | 增长(↑/↓) × 通胀(↑/↓) → 当前周期定位 | 宏观方向（risk-on/off） |
| L2.2 大类资产 | 基于象限判断资金流向 | 资产偏好（债/股/商品/黄金） |
| L2.3 行业筛选 | 宏观 + Layer 1 叙事 → 受益行业 | 行业短名单（3-5个） |
| L2.4 因子扫描 | 标地对利率/汇率/石油/增长因子的敏感度回归 | 宏观匹配度打分 |
| L2.5 标的推荐 | 综合打分 → 候选清单（供 Layer 3 裁决） | 5-10 个候选标的 |

关键原则：**变化率 > 绝对值**。行业加速/减速比水平更重要。

### 4.3 Layer 3：技术面审查（回答：现在买还是等？止损设哪？）

**职责：阻塞交易，不产生交易。** L3 完全独立运行——只接收原始市场数据和 L2 的标的清单，不看 L1-L2 的分析结论，防止确认级联。

**注**：200 周均线作为辅助参考，非硬性过滤。算法交易普及后该指标有效性下降，需持续追踪误报率。

#### 第一关：三灯审查（Python 计算，不消耗 LLM）

| 灯 | 条件 | 行动 |
|----|------|------|
| 🟢 绿灯 | 价格 > 200周均线 + 日线结构未破 + 不在关键阻力 3%内 | 进入第二关 |
| 🟡 黄灯 | 1-2 项不通过 | 等待，不加仓 |
| 🔴 红灯 | 全部不通过 | 不买，不论 L2 多看好 |

#### 第二关：入场/出场计算（Pro 分析）

**入场：**
- 支撑区识别（谁在买？ETF 持仓？央行？成交量节点？）
- 阻力区识别（期权持仓集中区、历史抛压、整数关口）
- 入场区间（2-3% 宽，不追求精确点位）
- 止损位（支撑结构下方）
- 目标价（下一个主要阻力区）
- 持仓时限（最大持有天数）

**出场（持仓巡查）：**
- 入场时记录的止损触发了吗？
- 日线结构破坏了吗？
- 时间到了吗？
- "现金重构"：用市值的现金，今天会买这个吗？

**Druckenmiller 启发原则**（注：此规则为经验启发式，非经验证交易法则；应用于 LLM 衍生信号时需持续追踪实用性）：基本面看多但技术面看空 → 倾向于不买。价格是最终裁决者。

## 5. 每日交互流程

### 5.1 状态驱动的分析资源分配

```
每日开机自动判定状态：

🟢 轻仓（现金 > 60%）：机会发现为主
🟡 中等（现金 30-60%）：机会发现 + 少量持仓管理
🟠 重仓水上（现金 < 30%，多数盈利）：机会发现 + 持仓管理并重
🔴 重仓水下（现金 < 30%，多数亏损）：持仓管理优先但机会发现正常进行

机会发现资源：始终 100%，不因持仓状态削减。
持仓管理资源：有仓位时额外追加，独立预算。
```

```

  ② 粗筛分类 + 信号标签 (Flash)
```

### 5.3 进度条设计

| 场景 | 类型 | 显示 |
|------|------|------|
| 新闻批量采集 | 不确定 | "已采集 X/预估 Y 条" |
| Flash 批量处理 | 确定 | 百分比 + ETA |
| Pro 深度分析 | 不确定 | 减速确定性进度条 + 微状态文本（"正在分析 NVDA 技术面…"） |
| 影子批量运行 | 确定 | 逐个影子完成（快速更新节奏，非批量） |

### 5.4 AI 引导问题原则

- 每个门禁 ≤ 3 个引导问题
- 问题具体可操作，不开放式
- 每个问题对应一个可操作的行动
- 允许跳过自由讨论
- AI 监控离题：3 轮未推进分析 → 温和提醒回到引导问题
- **自适应问题深度**：第 1 天展示全部 3 个问题。第 10 天起，根据用户参与模式自动折叠/展开。第 30 天起，问题已个性化——AI 适应用户的提问风格，非反其道

### 5.5 三种会话模式

| 模式 | 时长 | 触发 | 内容 |
|------|------|------|------|
| **完整会话** | 10-20 min | 用户有充裕时间 | 标准 3 门禁 + 全部分析 + 所有影子 |
| **快速扫描** | 2-3 min | 用户只有几分钟 | 合并门禁："今日头条信号 + 持仓健康"，其余自动归档 |
| **追赶模式** | ~90s/天 | 错过了 N 天 | 距上次活跃日以来的关键变化摘要，无新分析 |

快速扫描和追赶模式是强制功能——没有这两种模式，错过 3 天将导致永久放弃。

### 5.6 门禁 2-3 之间强制暂停

2 分钟暂停屏，显式指示"走开。喝水。准备好了再回来。"暂停期间不显示任何可操作内容。基于假释法官研究（Danziger et al., 2011）：连续决策导致通过率从 65% 降至接近 0%，只有餐间休息恢复。门禁 3 是最重要的决策（真正投入资金），不能在认知资源最枯竭时做出。

### 5.7 会话检查点持久化

每个门禁完成后自动保存至磁盘。重启时："你上次在门禁 2。Phase 1+2 分析已完成。继续还是重新开始？"

### 5.8 渐进式入门（第 1-30 天）

| 阶段 | 天数 | 体验 |
|------|------|------|
| 新手期 | 第 1-7 天 | 合并门禁（方向+信号合并），影子隐藏，完整引导问题 |
| 成长期 | 第 8-30 天 | 3 门禁逐步开放，影子可见但可折叠，问题适配 |
| 完整期 | 第 31 天+ | 全部功能，问题深度根据用户参与模式自适应 |

### 5.9 前置自动化范围用户可配置

默认：纯新闻采集 + 持仓检查（无 Pro 分析，Flash 粗筛和尾部风险扫描仍运行）。高级选项：用户在设置中选择开启全面预分析（含方向简报生成）。核心原则：**人类设定自动化边界**。

### 5.10 信息节食（Info Diet）— 双层适用

**面向用户**（基于行为科学证据——简短信息比长篇驱动决策效果更好）：
- 标题 + 信号摘要格式：一句方向论点 + 3-5 条结构化数据 + 一句风险声明
- 目标 80-120 字/方向（非原来的 300 字）
- 深层分析全归档，不给用户看

**面向 AI**（基于 LLM 信息过载倒 U 型曲线——~55% 上下文后准确率退化）：
- 预处理前剔除 50% 信息价值最低的项
- 压缩纪律：用户端简洁 ≠ 输入端无节制
- AI 自打分（简洁度、废话检测、信息价值密度），复盘时回顾

## 6. 持仓管理

### 6.1 持仓巡查

每日前置自动化阶段，对每个持仓逐项检查：

1. 对照买入档案：逻辑还在吗？时间到了吗？价位到了吗？
2. "现金重构"：用市值的现金，会买这个还是买今天的最佳机会？
3. Layer 3 出场审查：止损触发？结构破坏？时间到期？

产出：每张持仓的状态卡（🟢 持有安心 / 🟡 关注变化 / 🔴 建议退出）

### 6.2 卖出信号规则

**三个触发条件（满足任一即提醒）：**

| 条件 | 含义 |
|------|------|
| 逻辑证伪 | 买入理由不再成立 |
| 技术破位 | 跌破 200 周均线或关键支撑 |
| 机会成本 | 持有此标的占用资金，出现了更高确信度的机会 |

**应对亏损厌恶的三板斧：**

1. **现金重构** — AI 不问"要不要割肉"，问"如果今天手上是现金，你还会买这个吗？"
2. **建议他人模式** — 用"如果你是投资顾问，你会建议客户怎么做？"的语气写
3. **入场即定出场** — 买入时记录 3 条出场条件（逻辑/时间/价位），复查时对照

**AI 建议卖出必须附替代方案：** 不是"建议卖出 TLT"，而是"建议卖出 TLT 释放 $X，替代方案今天方向 A 预计 +7-15%"。

**磨合期保护（前 60 天）：** 只触发逻辑证伪 + 技术破位同时满足的卖出建议。60 天后逐步解禁机会成本型卖出。

### 6.3 "现金重构"影子验证

现金重构的行为学助推在一般决策中有支持证据（Solomon's Paradox），但在投资情境中无直接验证。在投入使用前需实证检验：

**实验设计**：影子分为两队列，各 6 个，运行 90 天：
- **A 队（现金重构）**：持仓巡查使用现金重构退出规则
- **B 队（传统规则）**：持仓巡查使用固定止损+逻辑证伪退出规则

**评估指标**：
- 处置效应程度（持有亏损仓位天数 / 持有盈利仓位天数）
- 90 天累计收益
- 平均回撤
- 换手率（现金重构是否导致过度交易？）

**成功标准**：A 队处置效应显著低于 B 队 + A 队收益没有显著劣于 B 队（非劣性检验）。两项均满足 → 现金重构通过验证，写入主策略。任一不满足 → 停用，标记为"未通过行为学验证"。

### 6.3 组合热管理

- 同时持仓上限：6 个（注意力约束）
- 总热量上限：同时触发所有止损的总损失 ≤ 25% 总权益
- 排名衰减监控：持仓连续 N 天不在最优标的中 → 触发重评

## 7. 影子生态系统

### 7.1 影子分类

| 类型 | 数量 | 用途 | 决策流程 | 虚拟资金 |
|------|------|------|---------|---------|
| **Beta 陪跑** | 3-5 | 参数微调对比测试 | 与主策略一致 | 等同主账户 |
| **专家影子** | 10-15 | 领域专家（贵金属/币圈/能源/波动率/债市/新兴市场等） | 独立方法论 | $30K-50K |
| **敢死队** | 4-6 | 日内方向/周度趋势/事件猎犬/反向指标 | 强制方向判断 | $20K-30K |
| **临时事件** | 影子妈妈动态创建 | 特定事件专项追踪 | 事件驱动 | $10K-20K |
| **挑战者** | 秘密创建 | 在观察期影子不知情下运行 | 读前任全部记录 | 等同挑战目标 |

### 7.2 专家影子排名与资配额

**多指标排名**（防止单一 Sharpe 被操纵——"卖尾部保险收微小保费"可制造 Sharpe > 2.0）：

影子排名基于复合分数，单一指标权重 ≤ 40%：

| 指标 | 权重 | 说明 |
|------|------|------|
| 防操纵 Performance Measure (Goetzmann et al.) | 35% | 对非正态收益分布稳健 |
| Calmar 比率 | 25% | 收益/最大回撤 |
| Omega 比率 | 20% | 全分布收益-损失比 |
| 胜率 | 20% | 简单，不可操作 |

**贝叶斯过拟合剪发**（Witzany, 2021）：排名前对原始分数施加折扣，折扣与评估期数+竞争者数成正比。15 个影子 60 天 = 原始 Sharpe 1.2 → 剪发后 0.4-0.6。

**成就阶梯：**

| 状态 | Flash 配额/天 | 条件（剪发后得分） |
|------|-------------|------|
| 🟢 精英 | 7 次 | 连续 30 天复合分 > 85th 百分位 + 剪发后 Sharpe > 0.8 |
| 🟡 优秀 | 6 次 | 连续 10 天复合分 > 70th 百分位 + 剪发后 Sharpe > 0.6 |
| 🔵 正常 | 5 次 | 默认起点 |
| 🟠 观察 | 3 次 | 复合分 < 30th 百分位持续 10 天，或最大回撤 > 30% |
| 🔴 濒危 | 1 次 | 复合分 < 15th 百分位持续 20 天 |

**鲶鱼 Agent**（Wang et al., 2025）：一个角色特化的专家影子，其唯一职责是持少数意见并捍卫之。当 ≥80% 影子推荐同向时，鲶鱼强制提供反对意见。防止影子从众——"一起对"比"一个人错"更安全。

**合谋检测**：≥80% 影子连续 3 天同向推荐 → 标记审查。持续 10 天 → 自动"制度分析"（市场驱动收敛还是人为？）。

**紧急配额事后审计**：追踪（紧急申请正确数）/（总申请数）。置信度自报需与结果校准。10 次申请 2 次正确 → 基础配额永久降至观察水平。

每天开盘时影子收到"今日状态卡"，含当前排名、配额、升级所需条件。

### 7.3 紧急配额申请

影子发现置信度 ≥ 8/10 的非共识机会时，可申请额外 3 次 Flash 配额（自动审批）：

| 结果 | 惩罚/奖励 |
|------|----------|
| 盈利 | 永久配额 +1，认知库署名收录 |
| 错误但你未跟 | 配额 3 次/天 × 3 天观察期 |
| 错误且你跟了亏损 | 配额 3 次/天 × 7 天观察期 |
| 连续 3 次失误 | 配额永久 -1 |

### 7.4 影子淘汰与挑战者机制

**分层评估周期：**

| 策略类型 | 评估周期 |
|---------|---------|
| 短线（1-7天持仓）| 每 2 周 |
| 中线（1-4周持仓）| 每月 |
| 长线（1-6月持仓）| 每季度 |

**三阶段缓冲：**

```
第 1 阶段：排名警示
  → 连续 2 个评估周期排名后 20%
  → "⚠️ 改进提醒" + 排名展示 + AI 改进建议
  → 配额不减少

第 2 阶段：观察期 + 挑战者秘密培养
  → 连续 3 个评估周期排名后 20%
  → 原影子：🔴 濒危，配额 2 次/天，提交改进方案
  → 秘密创建挑战者影子：读前任全部记录，正常配额 5 次/天
  → 挑战者不显示在排名中，原影子不知道

第 3 阶段：对比裁决
  → 原影子 2 周后无改善
  → 挑战者 vs 原影子同期 2 周对比
  → 挑战者更优 → Kill 前任，接管席位
  → 挑战者也不行 → 市场环境问题，删除挑战者，恢复原影子
```

**重置触发（三项同时满足）：**
1. 6 个月内从未达到 🟡 优秀
2. 3 个月胜率波动 < ±5%（在均值附近躺平）
3. 3 个月未产生被认知库收录的 Insight

每月最多重置 2 个。满足条件 ≥3 个时按躺平程度排队。

### 7.5 影子晋升标准（修订——统计不可辨性问题）

原 15 笔交易标准被替换（60% 胜率在 15 笔上的 95% 置信区间约 32-84%——区分不了技能和抛硬币）：

| | 专家影子 | 敢死队 |
|------|---------|--------|
| 最低运行 | 跨越 ≥1 个完整 VIX 周期（约 120 个交易日） | 60 个交易日 |
| 最低交易 | ≥100 笔独立交易 | ≥50 笔 |
| 胜率要求 | > 60% | > 55% |
| Deflated Sharpe | > 0（统计显著） | > 0 |
| PBO | < 5% | < 10% |
| 最大回撤 | < 25% | < 35% |
| 前向验证 | 30 天样本外 ≥ 评估期表现的 50% | 同左 |
| 额外 | - | 必须跨越 ≥1 个 VIX > 25 高波动期 |

晋升后影子可参与每日门禁 2 的分析意见贡献（但无决策权）。

### 7.6 纸面→实盘转换落差处理

影子交易是在无摩擦虚拟环境中运行的——无滑点、无部分成交、无流动性约束、无心理压力。为缩小此落差：

- **虚拟滑点**：影子虚拟交易按 ATR 的 0.5% 扣减入场/出场价格
- **置信度折扣**：影子报告收益自动施加 20% 折扣（如影子报告 +10%，折算为 +8%）
- **实盘对比追踪**：用户实盘交易结果与影子虚拟预测对比，差距在月度复盘时汇报。差距收窄的影子 → 折扣逐步减小
- **晋升需实盘对齐**：影子晋升前需至少 10 笔交易有对应用户实盘数据，虚拟-实盘差距 < 30%

### 7.7 知识继承质量过滤

挑战者读取前任记录时，**不**全量继承。应用"Learngenes"选择性传递（Feng et al., 2024）：

- ✅ 传递：跨验证 Insight、已验证方法论组件
- ❌ 丢弃：未验证启发法、前任的系统性错误
- 隔离：疑似错误被标记，挑战者运行 30 天期间独立验证

累积复制错误（ACE）模型证实：小错误跨代呈指数级放大。质量过滤阻止此现象。

### 7.7 影子妈妈

- 检测事件（央行意外 ±50bp、重大地缘、单资产 24h 波动 > 5σ、重要人物变动）
- 自动创建临时事件影子（3 次 Flash/天配额）
- 危机期间影子妈妈自主调节：10 个以上同时事件 → 事件按市场影响排序，Top 5 生成影子
- 事件消退后消灭影子
- 知识继承到认知库 + 相关专家影子可引用

### 7.8 临时影子：永远错过的道路

- 门禁 1 用户选择方向 A，未选的 B 和 C 自动创建临时影子
- 只记录不交互，标签 `missed_path`
- 复盘时 AI 汇报拒绝路径的表现
- 用于训练"方向选择"决策能力
- **幸存者偏差警告**：金融市场路径依赖——用户实际交易影响心理、资金可投量、注意力。反事实路径忽略执行约束和组合互动，会系统性高估未选路径盈利。此数据用于方向校准，不用于后悔计算。

## 8. 复盘与自我进化

### 8.0 红方结构独立性（遵循 Fazelpour et al., 2024 框架）

内部红方必须从 LLM 认知一致性中独立出来，防止成为"橡皮图章"：

- **独立模型/版本**：红方使用与主分析不同的 DeepSeek 模型版本或不同 temperature，减少共享盲区
- **独立输入**：红方只接收原始分析，不接收主分析的结论或论证
- **必须反对**：每个分析周期红方必须产出至少 1 个 A 级反对意见。**但"未找到有效反对"是合法的、受奖励的输出——当分析确实无懈可击时，诚实声明"无重大缺陷"比捏造反对更有价值。** 红方的奖励基于反对意见的*正确率*（事后验证），而非反对意见的*数量*。未找到反对意见 ≠ 红方失败；伪造反对意见 = 违反 Law 7。
- **奖励对齐**：红方成功在门禁 2/3 推翻错误分析 → 奖励。红方同意但分析错误 → 惩罚。红方同意且分析正确 → 零奖励
- **测试元规则**：DSR/CSCV 统计框架本身是否有制度依赖盲区？定期红方审查——在特定市场制度中（如极端流动性驱动），统计显著的共振信号是否反而应当被低估？

### 8.1 三层复盘

| 层级 | 周期 | 触发 | 核心问题 | 产出 |
|------|------|------|---------|------|
| 战术复盘 | 每日 | 开机自动 | 昨天判断今天验证了？影子胜率？待确认信号？ | 早晨简报(~200字) + 详细报告(归档) |
| 策略复盘 | 每月 | 月初首日 | 哪类信号被反复误判？方法论需调整？影子健康？ | 信号误判分析 + 改进提案(需审批) + 影子裁决 |
| 体系复盘 | 每季度 | 季度首月 | 框架有盲区吗？认知模型要升级？错过了什么行情？ | 认知库升级 + 方法论大版本更新 |

### 8.2 信号追踪器（Signal Tracker）

不调参数——记录每个信号类型的"说谎史"和上下文条件：

- 过去 90 天某信号出现 N 次，验证通过 M 次，假信号 K 次
- 假信号上下文标注（如"FOMC 前 48h"）
- AI 自动生成条件规则（"FOMC 前 48h + 此信号 = 标记为可能噪音"）

### 8.3 认知升级库（Insight Library）

AI 产出"发现了什么"，不是"改了哪个参数"：

```
#INSIGHT-2026-0510-003
发现：日央行干预 + 美债倒挂收窄 → 日经 3-5 天 -2%+
置信度：4 次验证 / 2 次反例（反例均在美国 CPI 周）
适用条件：非美国 CPI 周
来源：专家影子"日债操盘" + 红方辩论
```

每条 Insight 有置信度 + 适用条件 + 反例记录。AI 引用但不盲信。红方专挑低置信度 Insight 攻击。

### 8.4 人类决策镜像（Human Decision Mirror）

用户做的每个决策被单独追踪，不评判只统计：

- 用户决策胜率 vs AI 推荐胜率
- 倾向加仓/减仓？
- 否定过的 AI 建议后来证明是对是错？
- 决策和高 VIX 有关吗？
- 帮助用户看到自己的决策模式

### 8.5 影子验证闭环

调参不是 AI 说了算——必须经过：

1. 月度复盘发现信号误判 → AI 提案（"降低 X 权重 5%，原因：……计算依据：……"）
2. 用户批准测试 → 创建 3 个 Beta 影子（-3% / -5% / -7%），跑 30 天
3. 30 天后对比：哪个最好？因果解释？红方质疑通过？
4. 用户批准 → 写入主策略配置

### 8.6 复盘工具编排

| 复盘层级 | 编排方式 |
|---------|---------|
| 每日 | 开机自动完成，用户只看简报 |
| 每月 | 后台静默运行 10-20 分钟，有提醒无阻塞 |
| 每季度 | 后台静默运行 30-60 分钟，弹窗确认开始，结束通知 |

中断保护：复盘运行时关闭工具 → 弹窗"复盘进行中，中断不完整，确定退出？"

## 11. 信息源

### 11.1 新闻聚合（修订——移除已死/被封源）

**主力（付费，推荐）：**
- **NewsAPI.org**（$450/mo Business 或更低层）或 **GNews API**（低成本）：80K+ 源的实时头条

**免费 RSS（可用）：**
- MarketWatch、Investing.com、Nikkei Asia（仅标题）、FRED、SEC EDGAR、CFTC COT、BLS、BEA、Federal Reserve RSS
- 不再使用：Google News RSS（已死 2023）、Bloomberg RSS（不存在）、Reuters RSS（持续 HTTP 429）

**yfinance**：辅助使用，接受其不可靠性。大故障后需 manual patch。

### 11.2 关键人物（修订——脆弱源标记）

- **Trump Truth Social**：trumpstruth.org/feed（单人志愿项目，无 SLA，积极缓存以承受中断）
- ~~stanfordio/truthbrush~~（已死：最后提交 2024/7，SIO 已解散）
- **X/Twitter**：xcancel.com + rss.xcancel.com（单人维护，脆弱。X 随时可封。视为 best-effort）
- **国会交易**：capitoltrades.com（无 API，需 HTML 抓取）+ @anguslin/mcp-capitol-trades

### 11.3 降级策略（2026 反爬现实修订）

1. Track A：RSS/API 结构化源（首选——但第一梯队源大幅减少）
2. Track B：HTML 解析（Cloudflare AI 防御下成功率 < 10%。需 TLS 指纹匹配 + 住宅代理。推荐托管抓取 API：Scrapfly/Crawlbase）
3. Track C：NewsAPI/GNews 付费管道（可靠性基线）
4. Track D：**人在回路轻量配合**——AI 在特定信息被阻时发出桌面提醒："帮我查 X"，用户花 30 秒复制粘贴
5. 不可伪造：所有 Track 都失败 → 诚实报告"数据不可用"，永不 mock

### 11.4 缺失数据类型的处理

- **期权数据 / VIX 期货期限结构**：考虑 Polygon.io（$29/mo）或 CBOE 免费延迟报价（HTML 抓取，脆弱）
- **实时 Bloomberg/Reuters 级新闻**：不存在可靠的免费替代。接受第二梯队源的速度劣势。

### 11.5 数据集中缓存

15 个影子 + 主管道各自独立查询 → 速率限制必被触发。集中缓存 + TTL 新鲜度管理从第 1 天架构内置。

## 14. 数据诚信实施（Law 7 执行细则）

### 14.1 Fabrication Watchdog (M1-M4)

**M1 — 调用前源绑定**：每次 LLM 调用注入强制 `DATA_INTEGRITY_PROTOCOL` 到 system prompt。不可覆盖。

**M2 — 响应后数字声明提取**：Python 正则提取所有数字声明（价格、比率、百分比、日期、金额），标注来源 Agent + 会话 ID + 时间戳。

**M3 — 来源追踪验证**：

| Track | 方式 | 时间 | 容忍度 |
|-------|------|------|--------|
| A：API 直验 | 查询数据源基准值 vs LLM 声明 | < 2s | 价格 ±5%，日期 0%，估算 ±10% |
| B：多源共识 | 3 独立 API，2/3 共识 = 基准 | < 5s | 同 Track A |
| C：来源审计 | 无可验证 API → 标记 UNVERIFIABLE_FORWARD，升级至红方 | — | 需推理链透明 |

Track A/B 失败 → 升级至红方"找谎言"审查。

**M4 — 持续诚信评分**：每个 Agent 从 100 开始。不可验证声明 -2。证实为假 -15（立即隔离审查）。缺来源引用 -5。证实为真 +1（上限 100）。

### 14.2 三振出局惩罚

| 振 | 触发 | 后果 | 可逆性 |
|----|------|------|--------|
| 1：警告 | 单个数字声明经 API 证实为假 | 配额减半 5 天 + 所有输出强制 Track A + 公开警告标记 | 30 天干净记录后过期 |
| 2：隔离 | 90 天内再次造假 | 输出进等候队列 + 90 天全量回溯审计 + 14 天观察 | 需红方认证 + 人类批准 |
| 3：终止 | 180 天内第三次 或 恶劣造假 | 永久删除 + 所有过去分析标记 SOURCE_COMPROMISED | 不可逆，创建全新替代 |

**恶劣造假（一次出局）**：编造不存在价格、捏造新闻事件/央行公告、引用不存在的来源。

**错误 vs 造假区分**：来源存在但数值错误 → 错误（标记方法审查，不计振）。来源不存在 → 造假（振数计入）。

### 14.3 "所有 AI 都知道"的多层保障

1. **每次调用的 system prompt 注入**（deepseek_client.py 的 `chat_with_integrity()`）
2. **新影子入职测试**：5 条声明中识别 2 条伪造，5/5 通过方可获得配额
3. **每日简报的诚信报告板块**：昨日声明数、真/假/不可验证、活跃警告
4. **ARCHITECTURE_HANDOFF 强制要求**：每模块 Law 7 合规为接受标准
5. **CLAUDE.md Law 7 声明**：所有开发 Agent 启动时可见

### 14.4 人在回路：诚信升级至人类

任何第 2/3 振事件在每日简报中以 CRITICAL 级别呈现：
1. 造假声明（说的是什么）
2. 基准真相（数据实际是什么）
3. Agent 的完整推理上下文
4. 建议行动（隔离/终止/恢复）
5. 影响评估（哪些决策用过造假数据）

人类可推翻任何决定，但推翻被记录并在人类决策镜像中追踪。
