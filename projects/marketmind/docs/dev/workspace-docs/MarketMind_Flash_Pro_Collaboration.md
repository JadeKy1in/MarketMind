## 分工原则

| | Flash (DeepSeek Flash) | Pro (DeepSeek Pro) |
|------|------|------|
| **角色** | 筛选器 + 聚合器 | 思考者 + 决策者 |
| **处理量** | 587 条全部 | 10-15 个主题 |
| **深度** | 轻量（5轴打分+分类） | 深度（逻辑链+验证+对抗） |
| **输出** | 结构化JSON | 投资建议 |
| **Token** | ~60,000 | ~45,000 |

## Flash 的两个调用

### 调用 1: 轻量分流 (`flash_triage.py`)

**做什么**: 对全部标题打分+分类  
**输入**: 100 条标题/批  
**输出**: 
```json
{
  "headline_index": 0,
  "scores": {"market_impact": 7, "cross_source_corroboration": 5, 
             "contradicts_consensus": 3, "investigative_depth_needed": 6, "urgency": 8},
  "classification": "macro",
  "affected_assets": ["EUR/USD", "DAX"],
  "cluster_hints": ["eurozone", "monetary_policy"]
}
```

**成本**: 587条÷100条/批×~100token/条 = ~60,000 token

### 调用 2: 事件聚类 (待红方审核)

**做什么**: 把高分标题聚合成主题  
**输入**: 80-120 条高分标题的 cluster_hints + 标题  
**输出**: 10-15 个命名主题 + 跨主题因果链  
**成本**: ~15,000 token

## Pro 的自主思考

Pro 收到聚合主题后，**自行决定**调查什么、调用哪些API、深入多深。

**Pro 不需要 Flash 告诉它怎么做**——Flash 只负责筛选和聚合。Pro 的推理链：
1. "这三个主题(ECB、德国PMI、EUR/USD)之间可能存在因果关系"
2. "我先验证ECB的措辞是否真的鹰派→查利率期货定价"
3. "市场只定价了20%加息概率→我的假设有价值→继续验证"
4. "德国PMI超预期→经济基本面支持加息→假设增强"
5. "EUR/USD跌破1.05→市场还没反应过来→存在交易机会"
6. "强制看空: ECB可能只是口头干预,实际不加息→看空置信度45%<主假设60%×0.6→不降级"
7. "结论: EUR看涨,置信度0.81,进入Decision"

## Flash 不做的事

- ❌ 不做完整信号提取（那是旧 `flash_preprocessor.py` 的工作）
- ❌ 不做投资建议
- ❌ 不做逻辑推理
- ❌ 不调用外部API
- ❌ 不决定Pro调查什么

## Pro 不做的事

- ❌ 不阅读全部587条新闻（只看聚合主题下的代表文章）
- ❌ 不对每条新闻逐一分析
- ❌ 不使用Flash的评分作为投资决策依据（评分只是筛选用）
