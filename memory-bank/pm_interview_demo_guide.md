# PM 面试演示指南 — Cline OS Command Center V2.0

> 使用时机：今晚面试演讲前阅读。这不是技术文档，是对话大纲。
> 核心定位：面向非技术面试官，展示“系统工程的商业洞察力”。

---

## 1. 如何展示"设置中心"的实时热更新能力？

**推荐场景转化：面试官问 "你们的产品如何实现个性化配置？"**

### 你的标准话术（20 秒版）

> "我们的 Command Center V2.0 有一个设置中心，可以在不重启系统的情况下，**实时切换**界面字体、字号、配色方案和 API 密钥。
>
> 举个例子——如果风控团队习惯大字版 dark mode，交易台喜欢小字版 light mode，他们各自改完立即生效，不需要停服务。背后的机制是一个发布-订阅架构：保存时遍历所有已注册的订阅者，递归穿透整个 UI 树统一更新。"
>
> （注意：不要主动提 "SettingsManager.subscribe()" 或 "递归 widget 树" 这些技术细节，除非面试官追问）

### 如果你有笔记本现场演示

1. 启动系统：`python -m projects.command_center.app`
2. 点击右上角齿轮 → 设置弹窗
3. 在"外观"标签页，切换字体从 "Microsoft YaHei" 到 "Arial"
4. 滑动字号滑块从 14pt 到 20pt
5. 点击"保存设置" → 界面立即刷新字体和字号
6. 再切回 dark/light 模式，给面试官看视觉反差

### 能说的亮点

- **零中断**：字体/外观/字号的变更不重启、不闪退
- **亚秒级响应**：保存后立即刷新所有组件
- **多场景**：团队可各自调优，公共终端共享

## 2. 如何通过"附件功能"展示系统的多模态认知边界？

**推荐场景转化：面试官问 "你们的系统能处理哪些类型的数据？"**

### 你的标准话术（20 秒版）

> "我们的输入栏支持多种数据格式：图片、PDF、Markdown、纯文本、CSV。用户不需要写代码，直接拖入一份研报 PDF 或一张 K 线截屏，系统后台会自动 OCR 读取、提取文字、构建多模态消息，然后路由到对应的大模型处理。
>
> 这意味着我们的系统不仅能读文字，还能 '看懂' 图表和文档。这对金融场景特别有价值——很多关键信息是图片格式的，之前的系统只能处理文字，等于瞎了一半。"
>
> （注意：不需要解释 OCRReader 架构或 build_vision_messages 实现细节）

### 如果你有笔记本现场演示

1. 进入系统聊天面板
2. 点击输入栏左侧的 `[+]` 按钮
3. 选择一张截图（如 K 线图或一个 PDF 研报）
4. 输入 "请分析这张图中展示的趋势"
5. 点击"发送"
6. 系统后台自动 OCR 解析 → 提交到 Flash Vision API → 返回结构化回答

### 能说的亮点

- **零代码**：不需要写 SQL、不需要调 API，拖放即可
- **多模态**：同时支持文本 + 图片 + PDF 混合输入
- **异步不阻塞**：附件解析在后台线程，主界面继续流畅响应

## 3. 如何通过"一键战报"体现系统的商业价值？

**推荐场景转化：面试官问 "你们的系统做出了哪些可量化的输出？"**

### 你的标准话术（30 秒版）

> "我们有一个 '一键战报' 功能——用户点一下按钮，系统自动完成从新闻爬取到仓位优化、再到中文战报生成的完整链路，输出一份可导出 Markdown 或 PDF 的归因报告。
>
> 商业价值体现在三个层面：第一，原本需要 3 个分析师花一上午才能完成的工作，现在一键 3 秒搞定；第二，报告的句子全是自然语言，没有数学符号，可以直接发给客户或领导层看；第三，底层用了 Monte Carlo 影子对比，告诉你调整后的组合比原来好在哪里、风险变化了多少，不是拍脑袋的建议。

### 如果你有笔记本现场演示

1. 在 Dashboard 面板点击"一键战报"按钮
2. 状态栏依次显示：`正在拉取新闻` → `正在优化调仓` → `正在运行影子对比` → `正在生成报告`
3. 3-5 秒后，右侧对话面板展示完整的 Markdown 战报
4. 重点展示：中文自然语言描述（零数学符号）、信心等级（极强/较强/一般/较弱/极弱）、操作建议
5. 可选：点击"导出 Markdown"保存文件，展示给面试官

### 能说的亮点

- **全自动化**：从情报到优化到报告到导出，13 个模块 8 个 Phase 一步到位
- **中文自然语言**：没有 α/β/Var 等数学符号，PM 和领导层可读
- **可量化对比**：影子对比告诉你优化后的胜率、收益改善、风险变化
- **多格式导出**：一键导出 Markdown 和 PDF（优雅降级）

## 4. 备选问答（如果面试官深入追问）

### "你的系统用了什么技术栈？"

> "前端是 CustomTkinter（Python 的现代桌面 UI 框架），后端分三层：网关层（TaskQueue + 双模型路由）、智能情报层（爬虫 + 事实校验 + 信念修正）、引擎层（调仓优化 + Monte Carlo 影子对比 + 语义翻译 + 报告生成）。所有层都有 100% 测试覆盖。"

### "系统有多少测试？你的代码质量怎么保障？"

> "全量测试 881 项，所有阶段 100% 通过。每个模块有独立单元测试，还有 38 项端到端集成测试覆盖了从 Position 到 Report 的全链路。测试通过率是合入门槛，不是指标。"

### "你们的团队多少人？你在这个项目里负责什么？"

> （根据实际情况回答。你在这个项目中的角色是首席架构师，决策了 AD-001 到 AD-006 的全部架构方案，主编了主要模块的代码实现。）
